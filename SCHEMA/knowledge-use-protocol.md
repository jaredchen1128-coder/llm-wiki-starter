# 知识使用闭环协议

本协议定义知识库如何从“整理资料”进入“辅助提问、判断、决策和复盘”的闭环。

## 目标

知识库不能只收藏和整理，还必须被使用。

每次 Query、分析或决策后，都要判断：

- 回答是否产生长期价值。
- 是否需要写回主题页、概念页、问题页或决策页。
- 是否产生新的待研究问题。
- 是否改变已有结论。
- 是否暴露结构或语义问题。

## 1. Query 回写

满足任意一项，即应写回：

- 回答总结了可复用方法。
- 回答改变了某个核心结论。
- 回答提出了长期问题。
- 回答形成了决策依据。
- 回答发现页面过时、冲突、重复或缺少来源。

回写位置：

- 主题或概念：`WIKI/topics/`、`WIKI/concepts/`
- 长期问题：`WIKI/questions/`
- 真实决策：`WIKI/decisions/`
- 复盘和语义检查：`WIKI/reviews/`
- 操作摘要：`WIKI/log.md`

## 2. 问题库

`WIKI/questions/` 保存长期问题、开放问题和待研究问题。

创建问题页的条件：

- 问题会反复出现。
- 当前资料不足以回答。
- 问题跨多个主题。
- 问题影响后续决策。

## 3. 决策记录

`WIKI/decisions/` 记录知识库实际帮助做出的判断。

记录条件：

- 用户基于知识库选择了一个方案。
- 用户明确比较多个选项。
- 某次回答影响了行动计划、工作流程或长期策略。

## 4. 语义 Lint

结构 Lint 检查文件和链接是否完整；语义 Lint 检查内容是否仍然成立。

检查范围：

- 核心结论是否过时。
- 页面之间是否有观点冲突。
- 多个页面是否重复表达同一概念。
- 来源整理页是否真正影响了主题页或概念页。
- 长期问题是否无人处理。

结果写入 `WIKI/reviews/`，并追加 `WIKI/log.md`。
