# 知识迭代协议

本协议定义知识如何从原始资料进入知识库、如何更新旧知识、如何处理冲突、如何记录版本、如何复盘，以及如何归档废弃知识。

## 1. 新资料进入

新资料进入时，保持 `RAW` 原文不变，只在 `WIKI` 中沉淀理解。

步骤：

1. 定位新增资料所在的 `RAW` 路径。
2. 判断资料类型：教程、观点、案例、概念、项目记录、问题线索或参考资料。
3. 判断资料是否值得进入 `WIKI`。
4. 若值得进入，创建或更新 `WIKI/sources/` 来源整理页。
5. 更新相关主题页、概念页、问题页或决策页。
6. 更新 `WIKI/ingest-register.md`、`WIKI/index.md` 和 `WIKI/log.md`。

进入标准：

- 能复用到未来工作、学习、创作或决策。
- 能补充或修正已有主题。
- 能形成独立概念、方法或问题。
- 能连接两个以上主题。

## 2. 旧知识更新

新资料进入后，必须检查它对旧页面的影响：

- 是否补充旧页面。
- 是否修正旧页面。
- 是否反驳旧页面。
- 是否需要拆出概念页、问题页或决策页。
- `相关页面` 是否需要补充关系说明。

禁止：

- 直接覆盖旧结论。
- 删除仍有历史价值的旧观点。
- 添加无法解释关系的内部链接。

## 3. 冲突处理

冲突类型：

- 事实冲突。
- 判断冲突。
- 时间冲突。
- 语境冲突。

处理方式：

1. 在相关页面新增或更新 `冲突与修正` 小节。
2. 写清旧观点、新观点、来源和冲突类型。
3. 能判断优先级时，更新核心结论。
4. 不能判断时，写入 `待确认问题` 或 `WIKI/questions/`。
5. 在 `WIKI/log.md` 记录。

## 4. 版本记录

小变更记录在 `WIKI/log.md`。

重大修正、冲突处理、废弃归档应同时记录在页面内：

```markdown
## 版本记录

- YYYY-MM-DD：一句话说明变化。来源：`RAW/...`
```

## 5. 周期复盘

默认每 2 到 4 周做一次轻量复盘。

优先检查：

- 最近新增来源页。
- 高频更新主题。
- 没有来源的页面。
- 重复概念。
- 长期未解决问题。
- 可能过时的判断。

复盘结果写入 `WIKI/reviews/`，并追加 `WIKI/log.md`。

## 6. 废弃知识归档

废弃知识不直接删除，除非用户明确要求。

需要归档的情况：

- 内容明显过时。
- 判断被后续资料推翻。
- 页面主题重复，已合并到更好的页面。
- 页面价值很低，但仍需要保留历史记录。

归档到 `WIKI/archive/`，并从 `WIKI/index.md` 主入口移除。
